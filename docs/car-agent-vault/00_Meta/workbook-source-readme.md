# Workbook Source README

## 项目范围

这不是逐车型逐配置的销售参数表，而是一套“产品设计 / agent UX / ontology / vault”原型。它参考了 Tesla、小鹏、理想的官方手册/官网/OTA材料，以及你提供的场景分类图，产出一个适合继续扩写的知识结构。

## 使用方法

先看 01_Benchmark_Seeds，把真实产品能力抽成 capability family；再看 02~05 的 taxonomy；最后在 06~09 中把关系、workflow、优先级和 vault 落地。

## 关键原则

1) 安全优先于个性化；2) 偏好只作为策略先验，不突破安全包络；3) 学习尽量先在影子/回放里发生；4) 记忆要可见、可删、可控；5) 解释的目标是让用户预测系统下一步，而不是灌输系统有多聪明。

## 工作簿结构

01 基准能力种子；02 能力 taxonomy；03 用户行为 taxonomy；04 场景 taxonomy；05 证据/记忆/策略/HMI 节点；06 关系图谱；07 关键 workflow；08 信息优先级；09 Obsidian vault 映射。

## 注意

不同品牌功能会随车型、地区、硬件、订阅、软件版本 / OTA 变化。本 prototype 用“能力族”建模，而不是承诺任一车型当前一定具备全部功能。

## 图中参考已吸收的分类

手动驾驶、辅助驾驶升级/降级、退出类场景、特定智驾行为、地库/园区等弱图空间、路线视角/路况视角切换、制动与确认机制、HAVP/APA 搜路算路与用户制动等。
