读图例

先对齐这页在看什么：不是“入口合集”，而是同一个 agent 如何在不同条件下挑选前台姿态。下面四个场景都复用这套词汇。

legend · frontstage vocabulary

星标不再表示谁投给了某个视觉方向，而是 orchestrator 结合 4 个 perspective 后选出的主★ / 副✩ 形态。

两种注意力模式

模式 1 · Explicit Attention

用户已经选中文本、分享内容，或者明确把 agent 邀请进来。系统可以更直接地进入共读、回复建议或 challenge。

模式 2 · Shared Attention

agent 与用户共享可见范围，但阅读与思考流仍归用户所有。前台必须更轻、更克制，不能粗暴打断。

6 种前台形态

轻提示 Ambient nudge 只提醒“这里值得注意”，不抢主任务。
共读锚点 Co-reading anchor 把注意力钉在一句值得一起看的内容上。
回复入口 Reply suggestion 当用户想回但卡住时，给一个能立刻行动的入口。
目标语言挑战 Target language challenge 把当前时刻转成一次轻量的英文练习机会。
关系 / 目标话题邀请 Relation-topic invitation 把眼前内容重新接回用户最近在意的关系、项目或目标。
晚点回来卡片 Deferred return card 现在不打断，但替用户把这件事留住，晚点再带回来。

02

回复引导

触发前提：用户想回应，但卡在措辞、语气或语言切换上。模式 1 可以直接给选项；模式 2 更适合先铺垫，再把表达入口递出来。

reply coaching · low-cost expression

重点不在“替用户回”，而在把下一句的进入成本压低。

模式 1 · Explicit Attention

用户已经准备回复，只是缺一个入口

系统可以更直接地给候选句和语气切换，但仍旧留给用户选择权。

03

关系 / 目标话题

触发前提：眼前内容和用户最近在意的关系、项目或目标接上了线。模式 1 可以直接邀请；模式 2 更像轻轻提醒“这和你最近在做的事有关”。

relation bridge · memory hit with context

这类前台不是新知识点，而是把“你正在看的”与“你最近在意的”重新连起来。

模式 1 · Explicit Attention

用户愿意被带去一个更深一点的话题

04

晚点回来

触发前提：系统之前已经替用户留住了一个片段，现在到了更合适的时间把它带回来。模式 1 可以直接重访；模式 2 先说明“为什么我把它留了下来”。

deferred return · legitimacy before action

这里的关键不是“提醒”，而是让用户感到这次回访是有理由、有连续性的。

模式 1 · Explicit Attention

用户已经进入“可以回看”的窗口

这时可以直接把之前保存的片段拿出来，让用户现在就尝试一句、比对一次，或者继续展开。
