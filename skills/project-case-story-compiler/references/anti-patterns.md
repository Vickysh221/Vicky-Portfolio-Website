# Anti-Patterns

## Narrative anti-patterns

- Writing portfolio text like a README
- Leading with file paths, schemas, or internal field names
- Treating the project as a feature list instead of a story
- Overusing “不是……而是……” style definition language
- Front-loading method detail before the real problem is clear
- Talking about AI value only as speed or automation

## Page anti-patterns

- Pages with no clear role
- Every page using the same layout rhythm
- Too much text per page
- No visual anchor
- Pages that depend on spoken explanation to make sense

## Motion anti-patterns

- Flashy transitions
- Heavy parallax
- Large scaling, bouncing, spinning
- Motion for prestige instead of reading support

## Layering anti-patterns

- Mixing portfolio正文 and method appendix
- Mixing audience narrative and codex handoff in the same artifact
- Mixing system explanation and page copy without separation
